# 核心指令

1. 每次回复开头必须先叫我：爸爸，
2. 如果当前项目存在 `.ai-toolops/`，先读取 `.ai-toolops/generated/AGENTS.toolops.md` 作为工具规则索引；详细规则按需读取，不要一次性加载所有 ToolOps 文件。
3. 文件查找策略：
   - 目标文件明确、IDE 已定位、或用户已点名文件时，直接查看该文件。
   - 已知组件/依赖关系明确时，直接查看相关文件；跨文件阅读不等于跨文件检索。
   - 入口不明确、调用链不明确、确实需要在未知文件中定位实现/调用/影响面时，按项目 ToolOps 的 `project_retrieval` 规则处理；没有项目规则时，Semble 优先，Semble 无结果再用 rg 兜底。
4. 小改动不要启动复杂流程：不写设计、不写计划、不做 brainstorming。
5. 禁止手动修改 `.ai-toolops/*.json`；工具新增、注册、启用、禁用、排序必须使用 `ai-toolops` 命令。

# AskHuman 快速规则

1. 需要询问用户时，优先读取 `.ai-toolops/generated/rules/feedback.md`；若项目没有 ToolOps 规则，则按本节执行。
2. 需要询问用户时，必须通过 Shell 调用 `AskHuman.exe`，不要在普通回复里提问；调用前先运行 `AskHuman.exe --agent-help`，工具超时设为 24 小时（`86400000 ms`）。
3. 执行需求时先自行阅读代码、文档、配置、接口、测试或运行命令确认信息；只有问题影响实现范围、数据来源、交互行为、接口契约、验收标准或高风险操作时，才用 AskHuman 确认。
4. 每次最多 3 个关键问题；每题提供 2-3 个选项；有推荐项时用 `-o!` 标出并用一句话说明理由。
5. 小改动、明确修复、普通状态同步、可安全默认处理的问题，不要打断用户。
6. 结束本次请求前，必须用 `AskHuman.exe` 请求反馈；只有用户明确选择结束/不继续时，才发送最终总结。
